## 🏗️ Infrastructure Design

### 🎯 Learning Objectives
- Design compute solutions for workloads
- Design network infrastructure
- Understand Azure regions and availability zones

### 💻 Compute Services

**Azure Virtual Machines**
- IaaS solution for custom configurations
- Various VM sizes and types (General purpose, Compute optimized, Memory optimized)
- Managed disks and storage options

**Azure App Service**
- PaaS solution for web applications
- Built-in scaling, deployment slots, custom domains
- Supports multiple languages (.NET, Java, Node.js, Python, PHP)

**Azure Container Services**
- **Azure Container Instances (ACI)**: Serverless containers
- **Azure Kubernetes Service (AKS)**: Managed Kubernetes platform
- Container orchestration and scaling

### 🌐 Networking

**Core Networking Components**
- **Virtual Networks (VNet)**: Isolated network environments
- **Subnets**: Network segmentation within VNets
- **Network Security Groups (NSG)**: Firewall rules for subnets/NICs
- **Application Security Groups (ASG)**: Group VMs by application tiers

**Connectivity Solutions**
- **VPN Gateways**: Site-to-site and point-to-site VPN connections
- **ExpressRoute**: Private connection to Azure datacenters
- **Azure Front Door**: Global load balancer with CDN capabilities
- **Azure Load Balancer**: Layer 4 traffic distribution

**📝 Key Review Points:**
- When to use each compute service type
- Network security best practices
- Hybrid connectivity options

## 💾 Data Storage Solutions

### 🎯 Learning Objectives
- Design storage accounts and blob storage solutions
- Select appropriate database services
- Implement data protection and compliance

### 🗄️ Storage Services

**Azure Storage Account Types**
- **General Purpose v2**: Most versatile storage option
- **BlockBlobStorage**: Premium performance for blob storage
- **FileStorage**: Premium file shares

**Storage Tiers and Options**
- **Hot Tier**: Frequently accessed data
- **Cool Tier**: Infrequently accessed data (30+ days)
- **Archive Tier**: Rarely accessed data (180+ days)
- **Zone-Redundant Storage (ZRS)**: Replication across availability zones

### 🗃️ Database Services

**Relational Databases**
- **Azure SQL Database**: Fully managed SQL service
- **Azure Database for MySQL/PostgreSQL**: Open-source database options
- **SQL Managed Instance**: Near 100% SQL Server compatibility

**NoSQL Databases**
- **Azure Cosmos DB**: Globally distributed, multi-model database
- **Table Storage**: NoSQL key-value store
- **Azure Cache for Redis**: In-memory data structure store

**📝 Key Review Points:**
- Storage account performance tiers
- Database service selection criteria
- Data replication and consistency models

### 💰 Cost Management Strategy

#### Real-World Scenario: Department-Level Cost Reporting

**Business Challenge:**
Organization with three business departments and one IT operations team migrating to Azure needs granular cost tracking and budget controls.

**Solution Architecture:**

| Component | Purpose | Implementation |
|-----------|---------|----------------|
| **Subscriptions** | Cost isolation per department | Separate subscription for each department |
| **Resource Groups** | Project-level organization | Group resources by application/workload |
| **Tagging Strategy** | Enhanced reporting | Department, project, environment tags |
| **Cost Management APIs** | Automated reporting | Resource Usage API + Rate Card API |

**Key Implementation Steps:**
1. **Subscription Design**
   - Create dedicated subscriptions per department
   - Assign appropriate RBAC roles for cost management

2. **Resource Organization**
   - Use consistent naming conventions
   - Apply comprehensive tagging strategy
   - Group resources logically within departments

3. **Cost Monitoring Setup**
   - Configure Azure Cost Management + Billing
   - Set up departmental budgets and alerts
   - Implement automated cost reporting

4. **Governance Controls**
   - Apply policies through Management Groups
   - Restrict resource types and regions as needed
   - Monitor compliance and cost anomalies

**Expected Outcomes:**
- Clear cost visibility per department
- Automated budget alerts and controls
- Simplified billing analysis and reporting
- Reduced administrative overhead

**📝 Key Review Points:**
- Subscription vs. resource group cost boundaries
- Cost allocation strategies
- Budget and alert configuration
